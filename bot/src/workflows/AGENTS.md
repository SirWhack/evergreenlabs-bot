This scope defines the durable, long-running workflows that drive the bot's core data processing. It is responsible for orchestrating multi-step pipelines that react to external events (like GitHub webhooks) or scheduled triggers (like cron jobs), coordinating stateful operations across database reads/writes, LLM calls, and site data publishing. The workflows are built on Cloudflare Workers' `WorkflowEntrypoint` abstraction, which provides built-in step-based execution, retries, and persistence.

The primary entry points are two classes, both extending `WorkflowEntrypoint`. `PerRepoUpdate` handles per-repository webhook events. Its `run` method accepts a `PerRepoParams` object containing the repo name, delivery ID, and event type. It debounces rapid events using a private `DEBOUNCE_SECONDS` constant, drains pending commits via `drainPendingEvents`, fetches commit details from GitHub using `fetchCommitDetail`, and then runs the `draftLogEntries` pipeline to generate AI-drafted log entries. It also calls `introduceRepo` for repository metadata, `updateNow` for the "now" page, and finally `publishSiteData` to push the updated site. The `DailySync` workflow, triggered by a cron schedule, runs a broader set of pipelines: `runProjectSync` for GitHub project boards, `runRoadmapSync` for roadmap data, and `introduceOrphans` to handle unlinked entities. Both workflows use `getSitePart` and `putSitePart` from the state library to manage incremental site data.

The workflows depend on several internal scopes: the pipeline modules (`project_sync`, `roadmap_sync`, `log_drafter`, `introduce`, `now_updater`) for business logic, the `lib/state` module for database access, `lib/github` for API calls, `lib/commits` for commit extraction, and `lib/publish` for deploying the final site. Environment variables are typed through `DailySyncEnv` and `PerRepoEnv` interfaces, which include credentials for GitHub, OpenRouter, and D1 database bindings. This scope acts as the top-level coordinator, hiding the complexity of individual pipeline steps behind the workflow abstraction and ensuring reliable, debounced execution of the bot's data processing logic.
