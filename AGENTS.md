This scope defines the **evergreenlabs-bot**, a cloud-native automation system that watches public SirWhack GitHub repositories via webhooks, drafts log entries from commits using an LLM, and publishes them to the evergreenlabs website. The entire system runs on Cloudflare's serverless platform—Workers, Workflows, and D1 database—with no local machine dependency. The bot is deployed at `https://evergreenlabs-bot.swynnr.workers.dev` and is currently undergoing a phased migration from a Python codebase to TypeScript, documented in ADR 0001.

The public API surface consists of two HTTP endpoints. A **POST** to `/trigger/daily-sync` accepts an `Authorization: Bearer <token>` header to manually invoke the daily sync workflow. A webhook endpoint receives GitHub push events, which are debounced per repository before triggering the per-repo-update workflow. Authentication uses a pre-shared `TRIGGER_TOKEN` secret for manual triggers and a `GITHUB_WEBHOOK_SECRET` for webhook validation. GitHub App authentication (App ID 3841550, installation ID 135223603) uses pure Web Crypto instead of `@octokit/auth-app`, with installation tokens cached and refreshed with a 5-minute margin.

Internally, the bot orchestrates two Cloudflare Workflows. The **daily-sync** workflow (triggered at 03:00 UTC via cron or manually) runs: `project_sync` → `introduce` backstop → `roadmap_sync` → `publish`. The **per-repo-update** workflow (triggered by webhooks) runs: `introduce` → `log_drafter` (judge + draft) → `now_updater` → `publish`. Key code lives in `bot/src/`: `index.ts` handles routing, webhook processing, D1 dedup, and workflow triggers; `workflows/` contains workflow stubs; `lib/state.ts` provides D1 helpers; `lib/publish.ts` uses the GitHub Contents API to write `siteData.js`; `lib/github.ts` handles auth; `lib/llm.ts` is currently a placeholder that throws (to be filled in Slice 2). The `log_drafter` must filter out commits authored by the bot's own GitHub App to prevent infinite feedback loops.

The scope depends on several external services and infrastructure. **Cloudflare Workers** provides the execution environment, **D1** stores all mutable state (site_parts, drafts, cursors, webhook_dedup, pending_events, skipped_repos), and **Cloudflare Workflows** orchestrates the multi-step pipelines. **OpenRouter** (using `anthropic/claude-haiku-4.5`) handles all LLM calls. **GitHub** provides webhooks, the Contents API for publishing, and App authentication. Secrets (GITHUB_WEBHOOK_SECRET, GITHUB_APP_ID, GITHUB_APP_INSTALLATION_ID, GITHUB_APP_PRIVATE_KEY, OPENROUTER_API_KEY, TRIGGER_TOKEN) are stored in the user's `.env` file rather than `wrangler.toml`. The migration is sliced into 7 phases, with Slice 1 already merged (scaffold with tracer rows), Slices 2-3 running in parallel (log_drafter and daily-sync body), and Slice 7 being the human-in-the-loop cutover.
